Cithara es un proyecto en pleno desarrollo, cuyo objetivo es crear IAG para finales del año 2021, los retos que afronta este proyecto son muchos, una vez terminado el proyecto, tendrá múltiples usos: Chatbots, asistentes personales, asistentes científicos, motores de búsqueda, extracción de datos & oráculos, estoy en busca de financiadores que me proporcionen un sueldo, hasta terminar el proyecto mencionado.


Cithara versión actual?

Esta versión posee los principios elementales de un motor de visión abstracto para texto, esto permite reconocer la forma y posición relativa de cada carácter en una cadena de texto, para probar el máximo potencial posible, desarrolle el prototipo Cithara-I, el cual sirve para extraer caracteres obteniendo resultados parecidos a Regex, solo debes darle 2 o más cadenas de texto como ejemplos, para que el sistema aprenda a reconocer la composición & pueda extraer los datos deseados, luego debes darle un solo ejemplo para que sepa que debe extraer, bueno es hora de ver a Cithara en acción...



Tutorial de introducción

Para probar esta versión de Cithara haz clic en este botón Probar

Paso 1: para especificar mediante ejemplos como luce la cadena donde se van a extraer los caracteres deseados, solo debes escribir dos o más ejemplos, para hacer esto solo escribe el ejemplo deseado & presiona el botón sample

Paso 2: escribe el texto a extraer y presiona el botón. target

Paso 3: escribe algo que tenga similitud con el contexto que hayas creado & presiona el botón test o presiona la tecla ENTER, el sistema mostrara un mensaje con la predicción de caracteres que se pueden extraer de dicho contexto, repite este ultimo paso varias veces para probar sus limites & ventajas.

Borrar contexto: para crear un nuevo contexto solo debes hacer clic en el botón reboot

Making of Cithara-I

Esta versión de Cithara solo posee un motor abstracto para caracteres, cuyo objetivo es calcular la forma & posición relativa tanto del fondo como del dato a extraer, ¿como se calcula eso?: por medio de un algoritmo diferencial matemático, primero se convierte cada carácter a su correspondiente numero en la tabla decimal, luego se hace el mismo procedimiento pero esta vez con la tabla octal, luego se aplica el algoritmo matemático de forma iterativa fijándose solo en el siguiente o anterior carácter para unir las dos tablas en una sola devolviendo un rango de posiciones relativas donde puede posicionarse el carácter & un rango de posiciones donde se puede refactorizar el carácter original por el del rango, es decir el primer rango funciona para comprobar la posición del carácter en la cadena & el otro para saber que carácter mostrar en lugar del original, ¿como se extrae el texto de la cadena?...

Se itera el primer ejemplo del usuario hasta encontrar la primera concurrencia del texto a buscar dividiendo la cadena en 2 o 3 pedazos luego se usa el algoritmo matemático para convertir de nuevo la cadena pero teniendo en cuenta los demás ejemplos de dicha cadena & el dato a extraer, ¿como se compara un contexto con un Test?...

Cada contexto creado es simplemente una entrada convertida a un árbol donde cada token tiene su correspondiente atributo, cuando haces testing simplemente se usa el algoritmo matemático para convertir la nueva entrada a datos relativos teniendo en cuenta el dato a extraer y los demás ejemplos, luego se compara el nuevo árbol creado con el árbol del contexto, como ven el algoritmo tiene una lógica de funcionamiento discreta pero bien pensada, pero a nivel de calculo computacional es bastante ineficiente, aun así con un poco más de desarrollo permitirá avanzar hacia la segunda versión, ¿que tanto permite esto que Cithara avance?...

Las mejoras de los cálculos diferenciales, permitirá una mayor abstracción de los caracteres en la entrada del usuario, pero ademas ayudara a diferenciar contextos, es decir los N mensajes del usuario se codificaran con el mencionado algoritmo como si fuera una sola entrada, permitiendo extraer el contexto deseado por el usuario contando solo con un pocos ejemplos, el usuario tendrá que darle como mínimo 2 o tres ejemplos del contexto y el sistema sabrá que hacer, algo ni pensado en Machine-learning que necesita miles de ejemplos para aproximarse una respuesta, NOTA: este proyecto no usa redes neuronales ni Machine-learning, por ello tendrá en todas las versiones un resultado envidiable, por ejemplo esta versión puede funcionar correctamente en un ordenador domestico de baja gama como es mi caso.




Limitaciones

Memoria: cada vez que creas un nuevo contexto, la memoria de Cithara se reinicia, por ende el sistema solo reconoce un solo contexto, ya que es imposible reconocer N contextos sin tener una mapa conceptual, problema que estará resuelto en la próxima versión.

Objetivo: el único objetivo del sistema es encontrar los caracteres a extraer y convertir el input en una mascara predictiva, como ven el sistema demuestra su potencial, pero se queda corto para muchas cosas, la extracción y validado de datos involucra componentes lógicos & de ambigüedad, problema que estará resuelto para la próxima versión.

Etiquetado: posee solo dos opciones de etiquetado para extracción de datos, con lo cual el sistema queda a medio camino, perdiendo la oportunidad de usar miles de inferencias por cada carácter de forma bidireccional, este problema también estará resuelto en la próxima versión.

Heroku este sistema esta hospedado en Heroku, las cuentas gratuitas como es mi caso, posee limites de memoria RAM, por ende en cada ejemplo podrás escribir máximo 300 caracteres, para no usar un uso excesivo de la RAM, problema que estará resuelto cuando este proyecto tengo un inversor, de lo contrario las versiones posteriores ademas de otros problemas, tendrán una máximo de caracteres de menos capacidad pudiendo llegar incluso hasta menos de 100 caracteres, limitando notablemente la experiencia de usuario & potencial.

DIAGRAMA DE DESARROLLO


CITHARA I

Motor visual para reconocimiento abstracto de caracteres.

Cithara-I este prototipo puede extraer caracteres deseados en cadenas de texto con resultados muy superiores en algunas ocasiones al lenguaje Regex.

CITHARA II

Mejora del motor visual soporte de abstracción bidireccional.

Etiquetado contextual: en la versión anterior podías etiquetar un mensaje como sample o target, en esta versión podrás usar una mayor cantidad de etiquetas de 3 tipos distintos: Extracción de datos, lógica & lingüística, la idea es combinar lo mejor de la experiencia en una interfaz gráfica con la programación basada en ejemplos, hay conceptos humanos muy básicos que el cerebro humano lo tiene integrado como un pequeño lenguaje de programación, la consciencia aprende a diferenciar y reconocer cuando se debe activar cada herramienta de ese lenguaje, en el caso de Cithara no sera posible hacer eso ya que requiere un CPU capaz de procesar 8.000 millones de Zettabyte por nano segundo, hasta que todo el lenguaje de programación este representado por contextos & estados, por ello la otra opción rápida & eficaz es la interacción por interfaz gráfica donde el usuario pueda etiquetar los contextos desde un nivel muy bajo, diminuto & elemental, para luego poder reconocerlos y fusionarlos con operadores invisibles de forma automática en nueva información sin etiquetar.

Motor conceptual: convierte los mensajes etiquetados a un mapa conceptual bidireccional, también permite la fusión de una o más mapas como uno solo, permitiendo la generalización, NOTA: este motor le da ordenes que a aprendido de forma automática al motor visual para relacionar cada carácter de la extrada del usuario, permitiendo entender sin problemas el cáliz perdido del NLP: los errores ortográficos.

Cithara-II en la anterior versión solo podías crear un solo contexto de extracción, en esta nueva versión podrás crear la cantidad de contextos que desees con mayores opciones de etiquetado, el sistema reconocerá genéricamente cuando estas activando cada contexto & podrá auto-etiquetar nueva información que le proporciones, permitiendo por primera una comunicación amigable & lógica con la maquina donde ambos se ayudan mutuamente.

CITHARA III

Mejora exponencial del motor visual abstracción bidireccional & operadores lógicos invisibles.

Cithara-III esta versión podrá entender el lenguaje natural al 100% con temas donde no este embebido las emociones humanas, es decir las emociones quedaran fuera del campo abstracción, aun así podrás crear cualquier algoritmo matemático sin usar operadores, trabajar con problemas de entidades, relación y lógica, NOTA: el 100% de lo que expresamos en lenguaje natural tiene representado emociones, dichas emociones el sistema de Cithara no puede ni debe abstraer, por razones sociales & sexuales a largo plazo.

CITHARA IV

motor de introspección cuando una acción o contexto no se cumple permite crear un mapa conceptual inmutable.

Motor de atención: permite representar a lenguaje natural un mapa conceptual del motor de introspección & crear con nueva información un mapa conceptual mutable.

Cithara-IV básicamente permite que el sistema focalice su atención en un determinado error y lo represente en lenguaje natural como pregunta & repare el fallo con nueva y vieja información.

CITHARA V

Motor de abstracción permite la utilización de lógica difusa -> lógica abstracta.

Motor de trazado relacional: se encarga de ejecutar los mapas conceptuales con lógica difusa.

Cithara-V las implicaciones de esta ultima versión son cruciales, ejemplo: puedes enseñarle todo el conocimiento en: nano tecnología, física, medicina, biología, ingeniería genética y ingeniería viral, para decirle: creame un virus no mortal para curar personas con cáncer, eso es básicamente lo que hace convierte objetivos abstractos en absolutos.

CPUz: desarrollo extra que permite hacer un hackeo blanco a cualquier procesador para procesos paralelos, por ejemplo esto: permitirá la ejecución 157 procesos de gran consumo en un CPU de un solo núcleo de 2.80Ghz sin que que colapse el sistema, básicamente el hackeo permite cambiar el consumo de CPU en tiempo real en cada núcleo del CPU por proceso & priorizar su consumo, ejemplo: si N procesos tienen un progreso mayor a 50% se les da mayor capacidad de consumo, si un proceso termina el menor que N hereda su consumo, esta tecnología estará inspirada en las GPU Nvidia, tendrá uso en otros campos: Machine Learning & render 3D entre otros donde se pueda paralelizar un algoritmo, NOTA: no es un secreto que una PC usada de baja gama es mucho más barata que una GPU con las plataformas CUDA, otra ventaja es que al paralelizar un algoritmo de forma casi ilimitada en un CPU es que posee una gran gama de operadores lógicos a diferencia de una GPU que solo tiene un numero muy reducido de operadores.
